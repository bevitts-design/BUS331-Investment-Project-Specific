Overview
What Phase 3 Does

Phase 1 told you what the economy is doing. Phase 2 told you what your client needs. Phase 3 connects those two things mathematically. You convert your macro thesis into a set of return and risk assumptions for each asset class — the Capital Market Expectations (CME) Matrix — and then use Excel Solver to find the portfolio weights that maximize the Sharpe Ratio while staying within each client's IPS constraints.

→ The Connection Rule

Your CME Matrix numbers must be consistent with your Phase 1 macro thesis. If Phase 1 predicts high inflation, your Fixed Income expected return must be lower (bond prices fall as yields rise). If Phase 1 predicts a soft landing, equity returns should reflect moderate earnings growth. Graders will cross-check Phase 1 against Phase 3 directly.

Section 1
Building the CME Matrix
The CME Matrix is a table of expected returns and standard deviations for each asset class under your Base Case macro scenario. It is the mathematical translation of your Phase 1 thesis.

The workbook has two CME Matrix tabs — one for the Base Case and one for the Bear Case. You must complete both. The Base Case drives your Solver optimization. The Bear Case is used in the Phase 5 stress test.

Step 1 — Enter asset class assumptions

For each asset class, enter the expected annual return E(R) and standard deviation (σ) under your Base Case. These are not guesses — every number must be sourced:

  • U.S. Equities: Use FactSet @ECO consensus GDP and earnings growth to anchor your equity return. Historical average: 8–10% nominal, but adjust for your macro view.
  • Fixed Income: Current yield on 10-Year Treasury is a strong anchor. In a Late-Cycle environment with rates plateauing, expected bond returns are typically 4–6%.
  • Cash / Equivalents: Use the current Fed Funds Rate or 3-Month T-Bill yield (FRED: TB3MS). Cash return is stable but opportunity cost rises in bull markets.

Section 2
The Correlation Matrix
Correlation measures how much two asset classes move together. It is the key input that drives diversification — the lower the correlation, the more the portfolio benefits from holding both assets.

Your workbook has a 3×3 correlation matrix. The diagonal is always 1.00 (each asset is perfectly correlated with itself). The upper triangle is student input. The lower triangle auto-mirrors the upper triangle using formulas — never enter numbers there directly.

Base Case Correlations

Equities / Fixed Income (ρ₁₂): Historically –0.20 to +0.20. In a soft landing environment, typically slightly negative as bonds provide a modest hedge to equities.

Equities / Cash (ρ₁₃): Near zero (0.00 to 0.05). Cash is uncorrelated with market returns.

Bear Case Correlations ⚠

Critical Rule: In a Bear Case (market panic), correlations between risky asset classes converge toward 1.0. Diversification breaks down exactly when you need it most.

Equities / Fixed Income: Can shift from –0.20 to +0.70 or higher during a crisis (investors sell everything for cash).

Equities / Cash: Remains near zero — cash is a true safe haven.

This is why your Bear Case Sharpe Ratio is always lower than Base Case.

Section 3
Portfolio Variance & Standard Deviation
The portfolio standard deviation is not the weighted average of individual σ values. It must account for all pairwise correlations between asset classes. This is the core of Modern Portfolio Theory.

For a 3-asset portfolio with weights w₁, w₂, w₃ and standard deviations σ₁, σ₂, σ₃, the portfolio variance is:

σ²p = w₁²σ₁² + w₂²σ₂² + w₃²σ₃²
     + 2·w₁·w₂·ρ₁₂·σ₁·σ₂
     + 2·w₁·w₃·ρ₁₃·σ₁·σ₃
     + 2·w₂·w₃·ρ₂₃·σ₂·σ₃

σp = SQRT(σ²p)

-- This formula is already built into the cream cells in your workbook.
-- It pulls weights from Section 3, σ from Section 1, ρ from Section 2.
-- Do NOT hard-code numbers — the cells must update when you change inputs.

The workbook calculates portfolio σ and the Sharpe Ratio automatically once you enter the weights. You do not need to build a separate covariance matrix table — the formula above handles all the pairwise terms directly.

⚠ The 262% Error

The most common Phase 3 mistake is entering returns as whole numbers (e.g., 8.5 instead of 0.085) in the CME Matrix. This causes the GDP-style formula to produce nonsensical results like 262% growth. Always use decimal format: 8.5% = 0.085 in Excel formula cells. Yellow input cells are pre-formatted to display as % — enter 0.085 and it will show 8.5%.

Section 4
Solver Optimization — Maximum Sharpe Ratio
Excel Solver finds the portfolio weights that maximize the Sharpe Ratio — the highest return per unit of risk — subject to each client's IPS constraints. Every client will produce a different optimal allocation because their constraints differ.
What you need before running Solver
  • CME Matrix fully filled in (Section 1 and Section 2 complete)
  • Risk-Free Rate entered (FRED TB3MS — 3-Month T-Bill yield)
  • IPS max σ and max drawdown limits from Phase 2 for this client
  • Solver add-in enabled: File → Options → Add-ins → Excel Add-ins → Solver Add-in ✓
Solver Setup — Step by Step
Set Objective
Sharpe Ratio cell (col I, client's weight row)  →  MAX
By Changing
w₁, w₂, w₃ — the three weight cells (cols C, D, E for that client's row)
Subject To
w₁ + w₂ + w₃ = 1   (weights sum to 100%)
w₁ ≥ 0, w₂ ≥ 0, w₃ ≥ 0   (no short-selling)
Portfolio σ ≤ IPS max σ   (volatility constraint from Phase 2)
Solving Method
GRG Nonlinear  (required for Sharpe Ratio — it is a non-linear objective)
  1. 1
    Open the Excel CME Matrix workbook. Go to Data → Solver.
  2. 2
    In "Set Objective," click the Sharpe Ratio cell for your client (Section 3, column I). Select Max.
  3. 3
    In "By Changing Variable Cells," select the three weight cells (w₁, w₂, w₃) for that client's row.
  4. 4
    Click Add three times to set constraints: (1) weights = 1, (2) all weights ≥ 0, (3) portfolio σ ≤ IPS max σ.
  5. 5
    Select GRG Nonlinear as the solving method. Click Solve.
  6. 6
    When Solver returns a solution, select "Keep Solver Solution" and click OK. The weights now reflect the optimal allocation.
  7. 7
    Record the optimal weights, portfolio E(R), portfolio σ, and Sharpe Ratio. These carry forward into Phase 4 and Phase 5.
⚠ Hard-Coded Numbers = Zero Points

If a grader changes any input cell in your CME Matrix and the Sharpe Ratio, portfolio σ, and E(R) cells do not update, your model is hard-coded. This is the primary grading red flag for Phase 3. Every result cell must be a live formula — never type a calculated result directly into a cell.

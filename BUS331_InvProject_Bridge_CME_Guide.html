Build the Capital Market Expectations (CME) Matrix

One CME Matrix per client — justified by your Phase 1 macro thesis

The CME Matrix is the formal bridge between your macroeconomic forecast and your portfolio construction. For each asset class, you must specify an Expected Return and a Standard Deviation — and you must be able to defend both numbers using your macro thesis.

What "pre-populated with placeholder anchors" means

The values in the Anchor column below represent current long-run historical averages from Bodie, Kane & Marcus. Your team will override these with your own macro-justified estimates. A team forecasting a soft landing with rate cuts would lower Fixed Income yields and raise Equity returns relative to these anchors. A bearish team would do the opposite.

The Correlation Matrix — Required for Portfolio Math

To calculate portfolio standard deviation, Excel needs a correlation matrix in addition to the individual asset σ values above. Your team must populate this.

Bear Case Correlation Warning

In a Bear Case (market stress), correlations between asset classes — especially U.S. and International Equities — tend to move toward 1.0. This is a critical insight from Chapter 6 of BKM. A naïve Bear Case that uses normal correlations will understate the true portfolio risk. Your Bear Case correlation matrix should show tighter (higher) correlations than your Base Case.

2

Solver Optimization — Maximize the Sharpe Ratio

Find the asset allocation that delivers the best risk-adjusted return within IPS constraints

The goal is to find the portfolio weights (how much to allocate to each asset class) that produce the highest Sharpe Ratio, subject to the constraints defined in each client's IPS. This is the mathematical core of modern portfolio theory.

Sharpe Ratio Formula (Objective Function for Solver)
Sharpe Ratio = ( E(Rp) − Rf ) ÷ σp

Where:
  E(Rp) = SUMPRODUCT(weights, expected_returns)
  σp = SQRT( MMULT( TRANSPOSE(w), MMULT(CovMatrix, w) ) )
  Rf = Current risk-free rate (from Phase 1 — your Cash/T-Bill return)
⚠ The covariance matrix is built from the correlation matrix and individual σ values:
Cov(i,j) = ρ(i,j) × σ(i) × σ(j) — build this as a separate tab in Excel using named ranges.

Solver Setup — Step by Step

Step 1

Set Objective Cell

Set Objective to your Sharpe Ratio cell. Choose Max. This is the cell containing the formula above.

Step 2

Define Variable Cells

By Changing Variable Cells: select your 4–5 weight cells (e.g., B10:B14). These are the portfolio allocation percentages Solver will adjust.

Step 3

Add Constraints

  • Weights sum to 1.0 (100%)
  • Each weight ≥ 0 (no short selling, unless client IPS permits)
  • E(Rp) ≥ Client target return from IPS
  • σp ≤ Client max std dev from IPS
Step 4

Solver Method

Select GRG Nonlinear from the solving method dropdown. Click Solve. Accept the solution if Sharpe improves and constraints are met.

Simply changing numbers until Solver is happy is not acceptable.

3

Base Case vs. Bear Case Allocation

Run Solver twice — once per scenario — then compare

Every client portfolio requires two Solver runs: one using Base Case CME inputs, one using Bear Case CME inputs. This is not just a stress test — it's the foundation for the Phase 5 Audit. If the Bear Case allocation breaches the IPS drawdown limit, a re-allocation or hedge is required before presenting to the Investment Committee.

Feature Base Case Portfolio Bear Case Portfolio
CME Inputs Soft landing / consensus returns Red Zone: high rates, low sentiment
Correlations Normal long-run historical values Stressed — equities converge toward 1.0
Solver Objective Maximize Sharpe Minimize loss / check drawdown ≤ IPS limit
IPS Check E(R) and σ within IPS bounds? Max drawdown within IPS tripwire?
If it fails… Adjust weights and document why Re-allocate to cash or add hedge (Phase 4)

Required Excel Workbook Structure

One workbook per team — clearly organized for instructor review

Your Phase 3 Excel workbook must contain the following tabs. Hard-coded numbers anywhere in the model will cost points — every result cell must trace back to a formula referencing your CME inputs.

Tab 1

CME_Matrix

Base Case and Bear Case return/σ assumptions for all asset classes. One master table the entire model references. Include the macro justification column.

Tab 2

Covariance_Matrix

Built using: Cov(i,j) = ρ(i,j) × σ(i) × σ(j). Formulas must reference CME_Matrix cells. Separate tables for Base Case and Bear Case correlations.

Tab 3–5

Client_1, Client_2, Client_3

One tab per client. Contains: weight cells (yellow), portfolio return/σ formulas, Sharpe Ratio cell, IPS constraint checks, and the Solver setup. Base Case and Bear Case side by side.

Tab 6

Summary

Side-by-side comparison of all three client optimal allocations, Sharpe Ratios, and IPS compliance status. This is what the instructor reviews first.

⚠ Grading Red Flag — Static Tables

If your instructor changes a CME input (e.g., raises the U.S. Equity expected return by 1%) and the portfolio weights, Sharpe Ratio, and compliance checks do not update automatically, you have hard-coded your model. This is an automatic deduction. Every result must be formula-driven.

✓

Phase 3 Submission Checklist

Complete all items before submitting
  • CME Matrix complete — Base Case and Bear Case returns & σ for all asset classes, with macro justification column filled in
  • Macro link documented — Each CME assumption explicitly traces to a Phase 1 finding (GDP, CPI, Fed Funds rate forecast)
  • Correlation matrix built — Separate Base Case and Bear Case tables; Bear Case correlations are tighter (closer to 1.0) for equities
  • Covariance matrix formula-driven — Each cell uses Cov(i,j) = ρ × σ(i) × σ(j), referencing the CME and correlation tabs
  • Solver run for all 3 clients — Base Case (maximize Sharpe) and Bear Case (minimize drawdown) for each client
  • IPS compliance verified — E(R), σ, and drawdown limits checked against each client's IPS constraints from Phase 2
  • Bear Case failures corrected — Any portfolio that breaches the IPS drawdown tripwire has been re-allocated or flagged for Phase 4 hedging
  • No hard-coded numbers — Changing any CME input cell causes all dependent calculations to update automatically
  • Summary tab complete — All three clients shown side by side with final weights, Sharpe Ratios, and IPS status
